Changelog

Historial público resumido de cambios de Mapa Apagones, generado a partir de versiones curadas y commits recientes del repositorio.

v0.9.9.5-all-scope-polygons

  • Corrige el ámbito “Toda España” para usar polígonos municipales combinados de las comunidades disponibles.
  • Evita que zonas como Zuera o Madrid caigan a cuadrado/celda cuando ya existe GeoJSON municipal.
  • Mantiene el cuadrado solo como fallback o selección manual temporal cuando no hay polígono disponible.
  • Genera toda_espana_municipios.geojson desde datasets ya publicados, en formato ligero para no superar el límite de Cloudflare Pages.
  • No cambia backend, privacidad, Turnstile, rate limiting ni flujo de reportes.

v0.9.9.4-report-selection-state

  • Corrige selección de zonas activas usando una clave compatible con zone_id, id e incident_id.
  • Corrige clicks sobre polígonos/celdas activas también en modo reportar.
  • Evita que el botón “Reportar” borre una zona activa ya seleccionada.
  • Limpia estado visual cuando una zona queda resuelta o desaparece.
  • No cambia backend, privacidad, Turnstile, dependencias ni datasets geográficos.

v0.9.9.3-geo-aragon

  • Añade Aragón al selector geográfico.
  • Publica un GeoJSON municipal curado para Aragón.
  • Normaliza propiedades municipales para enlazar incidencias con polígonos.
  • Limpia el listado geográfico del README.
  • No cambia backend, privacidad, Turnstile ni lógica de reportes.
  • No sube el GeoJSON nacional bruto.

v0.9.9.2-geo-madrid-polygon-match

  • Normaliza el GeoJSON de Comunidad de Madrid para enlazar incidencias con polígonos municipales reales.
  • Añade municipio, province, zone_id y dataset_id a cada feature de Madrid.
  • Corrige la selección frontend para aceptar id, zone_id o incident_id.
  • No cambia backend, Turnstile, paquetes, privacidad ni flujo de reportes.
  • Mantiene “Toda España” sin cambio automático de ámbito.

v0.9.9.1-geo-madrid

  • Añade Comunidad de Madrid al selector geográfico.
  • Publica solo el GeoJSON municipal curado de Madrid.
  • No añade overlays nuevos ni cambia el flujo de reportes.
  • No toca backend, Turnstile, privacidad ni rate limiting.

v0.9.9.0-repo-hygiene-guardrails

  • Limpia el repositorio tras el rollback de estabilidad.
  • Elimina backups locales del control de versiones si se habían colado.
  • Refuerza .gitignore para backups, diagnósticos, logs, bases de datos, .env y GeoJSON brutos.
  • Añade guardas automáticas y CI básico para fallar antes del merge, no en producción.
  • Documenta un plan de estabilidad para rehacer geografía de forma incremental.

v0.9.8.9-stability-rollback

  • Rollback conservador al último frontend estable: v0.9.8.3-black-screen-fix.
  • Retira las capas experimentales de comunidades nuevas, selección municipal persistente y overlays problemáticos.
  • Recupera una base fiable para reportar, resolver y navegar sin pantalla negra.
  • Mantiene privacidad por diseño: sin cuentas, sin CUPS, sin texto libre, sin fotos y sin coordenadas exactas públicas.

v0.9.8.3-black-screen-fix

  • Corrige pantalla negra tras enviar o resolver un aviso.
  • Elimina una referencia JavaScript inexistente que podía romper React después del reporte.
  • Mantiene el overlay de reporte, pero evita que deje la app sin renderizar.
  • No cambia backend, privacidad, Turnstile ni lógica de reportes.

v0.9.8.2-report-overlay-stability

  • Unifica el overlay de reporte para que no parezca que se abren dos ventanas distintas.
  • Añade limpieza automática del overlay tras finalizar o si queda bloqueado.
  • Reajusta el mapa al cerrar avisos para evitar pantalla negra hasta refrescar.
  • No cambia backend ni la política de privacidad.

v0.9.8.1-report-actions-restore

  • Corrige acciones directas de incidencia: “Yo también” y “Ya volvió” ya no saltan al panel manual.
  • Mejora el reconocimiento de zona activa al marcar “Ya volvió” desde un punto cercano o visible.
  • Los errores de validación/verificación se muestran como aviso de error, no como éxito.
  • Reduce cambios bruscos entre overlay, panel y mapa durante el envío.

v0.9.8.0-report-overlay-flow

  • Mejora el overlay de reporte para explicar el flujo real de envío.
  • Cambia “detectando ayuntamiento” por validación de zona, protección anti-abuso, guardado anónimo y actualización de zona agregada.
  • Añade textos de ayuda más claros y menos alarmistas durante el envío.
  • Mantiene privacidad por diseño: sin CUPS, sin nombre, sin dirección exacta y sin texto libre.

v0.9.7.9-manual-restore-target

  • Corrige “Ya volvió” desde el modo Reportar manual cuando el punto cae sobre una zona activa.
  • El frontend deduce la incidencia visible y envía incident_id/zone_id cuando corresponde.
  • El backend añade fallback por bounds/celda activa antes de usar solo proximidad.
  • Evita repetir preflight después de Turnstile si ya se validó antes.

v0.9.7.8-report-flow-stability

  • Estabiliza el flujo de reporte, confirmación y “Ya volvió”.
  • Los cooldowns se comprueban antes del envío visual.
  • Mejora mensajes móviles y acciones visibles sin cambiar la política de privacidad.

v0.9.7.7-stable-rollback

  • Rollback de la pasada de banners que causaba regresiones móviles.
  • Recupera el flujo estable de reporte y acciones.
  • Mantiene Turnstile con fallback suave.
  • Corrige selección, acciones de incidencia, “Ya volvió” y mensajes duplicados.

v0.9.7.5-turnstile-soft-fallback

  • Hotfix de estabilidad para reportes si Turnstile queda bloqueado por el navegador.
  • Turnstile sigue activo, pero ya no deja el envío colgado.
  • Fallback a protección local y rate limiting del backend.
  • Evita estados permanentes como “Enviando…” o “Preparando…”.

v0.9.7.4-turnstile-managed

  • Hotfix de Turnstile para widget Managed/Gestionado.
  • Usa tamaño flexible o compacto según pantalla.
  • Mantiene comprobación al confirmar sin caja fija cortada.
  • Mejora compatibilidad en PC, iPhone/Safari y móviles pequeños.

v0.9.7.3-ios-turnstile-submit

  • Hotfix del flujo de reporte en iPhone/Safari.
  • El botón Confirmar reintenta preparar Turnstile invisible si todavía no está listo.
  • Si la verificación no responde, el botón se libera y aparece un mensaje claro.
  • Mantiene API pública, Turnstile invisible y flujo de reporte sin caja visual.
